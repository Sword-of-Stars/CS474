\vspace{1em}
\noindent\textbf{Understanding the Indis Table:}
\begin{itemize}[itemsep=0.2em]
\item \textbf{Empty cells} correspond to a \texttt{True} value, meaning we have not yet learned that the pair of states are distinguishable. These pairs are \emph{potentially indistinguishable}.
\item \textbf{Cells marked with ``X''} correspond to a \texttt{False} value, indicating we have determined that the pair of states \emph{are} distinguishable. There exists at least one input string that leads to different outcomes (one accepts, one rejects) from these two states.
\item The table only shows cells below the diagonal because:
  \begin{itemize}
  \item Indis$(p, q)$ = Indis$(q, p)$ (order doesn't matter)
  \item Indis$(p, p)$ is always \texttt{True} (a state can't be distinguished from itself)
  \end{itemize}
\end{itemize}

\section{Make the Resulting Minimized DFA}

We must now merge the indistinguishable states and determine the start state, transition function, and set of accepting states ``in the standard way.''

\subsection*{Identifying Indistinguishable States}

After the algorithm terminates, all \textbf{empty cells} in the Indis Table (those with \texttt{True} values) represent pairs of states that cannot be distinguished by any input string. These states behave identically from the perspective of language acceptance.

\textbf{Key insight:} If Indis$(p, q)$ = \texttt{True} at the end, then for \emph{every possible input string} $w$, either both $p$ and $q$ accept $w$ or both reject $w$. Since they're functionally equivalent, we can merge them into a single state.

\item States $\{ {{ block|join(", ") }} \}$ merge into state ${{ block|join("") }}$

\subsection*{Constructing the Minimized DFA}

\begin{enumerate}[itemsep=0.3em]
\item \textbf{States:} One state per equivalence class
\item \textbf{Start state:} The merged state containing the original start state
\item \textbf{Accepting states:} A merged state is accepting if any of its component states was accepting in the original DFA
\item \textbf{Transition function:} For each merged state and input symbol, pick any representative from the equivalence class, see where that state transitions, and the merged state transitions to whichever merged state contains that target
\end{enumerate}

The minimized DFA accepts exactly the same language as the original DFA but uses the minimum number of states possible. States that could not be distinguished by any input string have been merged while preserving the transition structure and accepting behavior.

\subsection*{Why This Works}

\begin{itemize}[itemsep=0.3em]
\item \textbf{Correctness:} Two states are merged only if no input string can distinguish them (empty cells in Indis Table)
\item \textbf{Completeness:} The algorithm exhaustively checks all possible distinguishing strings through iterative refinement
\item \textbf{Termination:} Guaranteed because cells can only change from \texttt{True} to \texttt{False} (never the reverse) and there are finitely many cells
\item \textbf{Minimality:} The result has the fewest possible states while preserving the language — merging any states that are marked as distinguishable would change the language
\end{itemize}
